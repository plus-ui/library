The Popconfirm component displays a small overlay confirmation dialog next to a target element, requiring the user to confirm an action. It's useful for actions like deleting items or submitting forms.

## Usage

### Basic Example

A basic popconfirm triggered by clicking a button.

<code-bubble class="not-content">

```html
<plus-popconfirm title="Delete Item?" confirm-text="Yes, Delete" cancel-text="No">
  <div slot="content">This action cannot be undone.</div>
  <plus-button status="danger">Delete</plus-button>
</plus-popconfirm>
```

```tsx
import { PlusPopconfirm, PlusButton } from '@plusui/library/react';

export default () => {
  return (
    <PlusPopconfirm title="Delete Item?" confirmText="Yes, Delete" cancelText="No">
      <div slot="content">This action cannot be undone.</div>
      <PlusButton status="danger">Delete</PlusButton>
    </PlusPopconfirm>
  );
};
```

## Accessibility (a11y)

*   **Keyboard Behavior:**
    *   When the popconfirm is opened, focus is typically moved inside the dialog, usually to the first focusable element (like the Cancel or Confirm button).
    *   `Tab` and `Shift+Tab` navigate between focusable elements within the popconfirm (title, content if focusable, Cancel button, Confirm button, Close button if present). Focus is trapped within the dialog.
    *   `Escape` closes the popconfirm and returns focus to the trigger element.
    *   `Enter` or `Space` activates the focused button (Confirm, Cancel, Close).
*   **Screen Reader:**
    *   The component uses `role="dialog"`, `aria-modal="true"`, `aria-labelledby` (pointing to the title element), and `aria-describedby` (pointing to the content element).
    *   When opened, screen readers announce the dialog's role, title, and description.
    *   Buttons (Confirm, Cancel, Close) have accessible names derived from their text content or `aria-label` if using custom slots.
*   **Required Developer Actions:**
    *   Ensure the trigger element (in the default slot) is inherently focusable or has `tabindex="0"`.
    *   Provide clear and concise text for the `title` prop or `title` slot.
    *   Provide descriptive content in the `content` slot.
    *   If using custom content in `title` or `content` slots, ensure it's accessible.
    *   Use appropriate `status` values to convey semantic meaning.

## API Reference

### Props

| Name         | Type                                                                                             | Default            | Description                                                         | Required |
|--------------|--------------------------------------------------------------------------------------------------|--------------------|---------------------------------------------------------------------|----------|
| `size`       | `'sm' \| 'md' \| 'lg'`                                                                           | `'md'`             | Size of the popconfirm dialog.                                      | No       |
| `orientation`| `'top' \| 'top-start' \| 'top-end' \| 'bottom' \| 'bottom-start' \| 'bottom-end' \| 'left' \| 'left-start' \| 'left-end' \| 'right' \| 'right-start' \| 'right-end'` | `'top'`            | Position relative to the trigger element.                           | No       |
| `trigger`    | `'click' \| 'hover'`                                                                             | `'click'`          | How the popconfirm is opened.                                       | No       |
| `title`      | `string`                                                                                         | `'Are you sure?'`  | Title text displayed in the header. Use `title` slot for custom HTML. | No       |
| `confirmText`| `string`                                                                                         | `'Confirm'`        | Text for the confirm button.                                        | No       |
| `cancelText` | `string`                                                                                         | `'Cancel'`         | Text for the cancel button.                                         | No       |
| `status`     | `'success' \| 'warning' \| 'danger' \| 'info' \| 'primary' \| 'default'`                         | `'default'`        | Color scheme and default icon based on status.                      | No       |
| `statusIcon` | `boolean`                                                                                        | `true`             | Whether to show the default status icon.                            | No       |
| `showArrow`  | `boolean`                                                                                        | `true`             | Whether to display the arrow pointing to the trigger element.       | No       |

### Events

| Name                      | Payload Type        | Description                              |
|---------------------------|---------------------|------------------------------------------|
| `plus-popconfirm-open`    | `CustomEvent<void>` | Fired when the popconfirm opens.         |
| `plus-popconfirm-close`   | `CustomEvent<void>` | Fired when the popconfirm closes.        |
| `plus-popconfirm-confirm` | `CustomEvent<void>` | Fired when the confirm button is clicked. |
| `plus-popconfirm-cancel`  | `CustomEvent<void>` | Fired when the cancel button is clicked.  |

### Slots

| Name              | Description                                                      |
|-------------------|------------------------------------------------------------------|
| `default` (unnamed) | The target element that triggers the popconfirm.                 |
| `icon`            | Custom icon content, replaces the default status icon.         |
| `title`           | Custom title content, overrides the `title` prop.                |
| `header`          | Custom header content, replaces the entire default header section. |
| `footer`          | Custom footer content, replaces the entire default footer section. |
| `content`         | The main descriptive content inside the popconfirm body.         |

### CSS Shadow Parts

| Part         | Description                             |
|--------------|-----------------------------------------|
| `popconfirm` | The main popconfirm container element.  |
| `arrow`      | The arrow element pointing to the trigger. |
| `header`     | The header section container.           |
| `title`      | The title element container.            |
| `content`    | The main content area container.        |
| `footer`     | The footer section container.           |
