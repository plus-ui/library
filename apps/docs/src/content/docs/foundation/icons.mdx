# Icons

Plus UI provides a unified SVG icon system that delivers consistent performance and rendering across all components. Our icon system features a comprehensive library of optimized SVG icons with size variants and simplified API.

## Usage

### Basic Icon Usage

```html
<plus-icon iconName="user"></plus-icon>
<plus-icon iconName="home"></plus-icon>
<plus-icon iconName="heart"></plus-icon>
```

### Size Variants

Icons support multiple size variants for different use cases:

```html
<plus-icon iconName="user" size="xs"></plus-icon>  <!-- 0.75rem -->
<plus-icon iconName="user" size="sm"></plus-icon>  <!-- 0.875rem -->
<plus-icon iconName="user" size="md"></plus-icon>  <!-- 1rem (default) -->
<plus-icon iconName="user" size="lg"></plus-icon>  <!-- 1.25rem -->
<plus-icon iconName="user" size="xl"></plus-icon>  <!-- 1.5rem -->
```

### Custom Styling

Icons inherit color from their parent and can be styled with CSS:

```html
<plus-icon iconName="heart" style="color: red; --icon-size: 2rem;"></plus-icon>
```

## Migration from FontAwesome

The Plus UI icon system replaces FontAwesome with optimized SVG icons:

### Before (FontAwesome)
```html
<plus-icon iconName="user" prefix="fas"></plus-icon>
<i class="fa-solid fa-user"></i>
```

### After (SVG Icons)
```html
<plus-icon iconName="user"></plus-icon>
```

## Performance Benefits

- **Smaller Bundle Size**: ~50KB reduction compared to FontAwesome
- **Local-First**: No external dependencies or CDN requests  
- **Optimized SVG**: Lightweight, scalable vector graphics
- **Consistent Rendering**: Unified styling across all components

## CSS Custom Properties

Customize icon appearance using CSS custom properties:

```css
plus-icon {
  --icon-size: 1.5rem;      /* Custom size */
  --icon-color: #3b82f6;    /* Custom color */
}
```

## Accessibility

Icons automatically include proper accessibility features:

```html
<!-- Decorative icons -->
<plus-icon iconName="heart" aria-hidden="true"></plus-icon>

<!-- Meaningful icons -->
<plus-icon iconName="user" role="img" aria-label="User profile"></plus-icon>
```

## Component Integration

Icons work seamlessly across all Plus UI components:

```html
<!-- Avatar with icon fallback -->
<plus-avatar icon="user"></plus-avatar>

<!-- Button with prefix icon -->
<plus-button>
  <plus-icon iconName="plus" slot="prefix"></plus-icon>
  Add Item
</plus-button>

<!-- Alert with status icon -->
<plus-alert status="success">
  <plus-icon iconName="check" slot="icon"></plus-icon>
  Operation completed successfully
</plus-alert>
```

## Best Practices

- Use **semantic icon names** that clearly indicate their purpose
- Prefer **size variants** over custom CSS sizing for consistency
- Include **aria-labels** for icons that convey meaning
- Use **aria-hidden="true"** for purely decorative icons
- Test icon visibility across different themes and color schemes
