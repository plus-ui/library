The Badge component is used to display small pieces of information, such as notification counts or status indicators, typically attached to another UI element.

## Usage

### Basic Example

A basic badge can display content (like a number) attached to an element, or act as a simple status dot.

<code-bubble class="not-content horizontal">
```html
<!-- Badge with numeric content -->
<plus-badge content="5">
  <plus-avatar image="https://i.pravatar.cc/50" alt="User Avatar"></plus-avatar>
</plus-badge>

<!-- Dot badge indicating status -->
<plus-badge status="success">
  <plus-avatar image="https://i.pravatar.cc/50" alt="User Avatar"></plus-avatar>
</plus-badge>
```

```tsx
import { PlusBadge, PlusAvatar } from '@plusui/library/react';

export default () => {
  return (
    <>
      {/* Badge with numeric content */}
      <PlusBadge content="5">
        <PlusAvatar image="https://i.pravatar.cc/50" alt="User Avatar" />

      <PlusBadge status="success">
        <PlusAvatar image="https://i.pravatar.cc/50" alt="User Avatar" />
      </PlusBadge>
    </>
  );
};
```

### Placement & Offset

Use the `orientation` prop to place the badge at different corners (`top-right` (default), `top-left`, `bottom-right`, `bottom-left`) relative to the element in the default slot. Use the `offset` prop (format: "x,y") to fine-tune the position.

<code-bubble class="not-content horizontal">
```html
<plus-badge orientation="top-left" content="TL">
  <plus-avatar image="https://i.pravatar.cc/50"></plus-avatar>
</plus-badge>
<plus-badge orientation="bottom-left" offset="-5px,5px" content="Offset">
  <plus-avatar image="https://i.pravatar.cc/50"></plus-avatar>
</plus-badge>
<plus-badge orientation="bottom-right" content="BR">
  <plus-avatar image="https://i.pravatar.cc/50"></plus-avatar>
</plus-badge>
```

## Accessibility (a11y)

*   **Keyboard Behavior:** Badges are typically non-interactive and do not receive focus.
*   **Screen Reader:**
    *   The component uses `role="status"` and `aria-live="polite"` to ensure that badge content (especially counts or status changes) is announced by screen readers.
    *   When a badge represents critical information (e.g., "Offline"), ensure the surrounding context or the element the badge is attached to provides sufficient information. For purely decorative badges or status dots where the meaning is clear from context (e.g., a green dot next to "Online"), explicit labeling might not be necessary, but consider if additional `aria-label` or `aria-describedby` on the associated element could improve clarity.
*   **Required Developer Actions:**
    *   If the badge conveys important status information not otherwise available textually (e.g., a red dot indicating an error without accompanying text), provide an accessible label for the element the badge is associated with, or use visually hidden text within the host element to describe the badge's meaning.

## API Reference

### Properties

| Name         | Type                                                              | Default        | Description                                                         | Required |
|--------------|-------------------------------------------------------------------|----------------|---------------------------------------------------------------------|----------|
| `status`     | `'default' | 'primary' | 'success' | 'warning' | 'danger' | 'info'` | `'default'`    | Color variant of the badge.                                          | No       |
| `size`       | `'sm' | 'md' | 'lg'`                                          | `'md'`         | Size of the badge.                                                   | No       |
| `orientation`| `'top-left' | 'top-right' | 'bottom-left' | 'bottom-right' | ''` | `'top-right'`  | Position relative to the default slot content. Leave empty for inline. | No       |
| `offset`     | `[string, string]` (parsed from "x,y")                       | `undefined`    | Fine-tune position with x,y offset (e.g., "5px,-2px").           | No       |
| `content`    | `string`                                                          | `undefined`    | Text/number content for the badge (alternative to slot).             | No       |

### Slots

| Name            | Description                                                        |
|-----------------|--------------------------------------------------------------------|
| (default)       | The element the badge is attached to (anchor element).             |
| `badge-content` | The content displayed inside the badge (overrides `content` prop). |
