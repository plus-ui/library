Our Radio components provide an intuitive and accessible way for users to select a single option from a set of choices. Designed for mutual exclusivity, the Radio component shines when organized within a RadioGroup, allowing for clear, organized selection in forms, settings, and preference controls.

## Anatomy

The Radio component features a circular button with a distinctive selection indicator and clear label, creating a user-friendly and visually appealing interface for option selection.

<Image src={anatomy} alt="anatomy of radio component" />

## Properties

Radio buttons can be customized with various properties to match your interface needs. From size variations to different states, our Radio component offers flexibility while maintaining consistent behavior.

| Checked | State    | Size   | Error   | Radio Group |
| ------- | -------- | ------ | ------- | ----------- |
| True    | Default  | Small  | Default | Horizontal  |
| False   | Readonly | Medium | Error   | Vertical    |
|         | Disabled | Large  |         |             |

### Checked

The checked state indicates the user's selection within a group of options:

* **Unchecked radio** represents an unselected option, ready for user interaction
* **Checked radio** shows the currently selected option with a filled circle

Radio buttons have different states to communicate their interactivity to users:

* **Default state** is interactive and ready for selection

### Form Integration

The RadioGroup component seamlessly integrates with forms, providing automatic value handling for easy form submissions. When a radio option is selected, its value is accessible both for form submission and for dynamic UI updates.

<code-bubble class="not-content">

```html
<div id="radio-form-example">
  <form id="preferences-form">
    <plus-radio-group name="theme" label="Select a theme" required>
      <plus-radio value="light" text="Light" checked></plus-radio>
      <plus-radio value="dark" text="Dark"></plus-radio>
      <plus-radio value="system" text="System"></plus-radio>
    </plus-radio-group>
    <div class="result-display">Selected theme: <span id="selected-theme">light</span></div>
    <button type="button" id="submit-button">Save Preferences</button>
  </form>

  <div id="submission-result" class="submission-result"></div>

  <script>
    /* Global function that runs immediately when the script loads */
    (function() {
      /* Get DOM elements */
      const form = document.getElementById('preferences-form');
      const radioGroup = form.querySelector('plus-radio-group[name="theme"]');
      const resultDisplay = document.getElementById('selected-theme');
      const submitButton = document.getElementById('submit-button');
      const submissionResult = document.getElementById('submission-result');
      
      /* Initialize with the default checked value */
      if (resultDisplay && radioGroup) {
        resultDisplay.textContent = radioGroup.value || 'light';
        
        /* Update the display when selection changes */
        radioGroup.addEventListener('plus-group-change', (event) => {
          resultDisplay.textContent = event.detail.value;
        });
      }
      
      /* Add click event handler to the submit button */
      if (submitButton) {
        submitButton.addEventListener('click', () => {
          /* Get the form data */
          const formData = new FormData(form);
          const theme = formData.get('theme');
          
          /* Display the submission result */
          if (submissionResult) {
            submissionResult.textContent = `Form submitted with theme: ${theme}`;
            submissionResult.style.display = 'block';
          }
          
          console.log(`Form submitted with theme: ${theme}`);
        });
      }
    })();
  </script>

  <style>
    .result-display, .submission-result {
      margin: 10px 0;
      padding: 8px;
      background-color: #f0f0f0;
      border-radius: 4px;
      font-family: monospace;
    }
    .submission-result {
      background-color: #e6f7e6;
      display: none;
    }
  </style>
</div>
```

```tsx
import { PlusRadioGroup, PlusRadio } from '@plusui/react';
import { FormEvent, useState } from 'react';

const PreferencesForm = () => {
  const [selectedTheme, setSelectedTheme] = useState('light');
  
  const handleChange = (event: CustomEvent) => {
    setSelectedTheme(event.detail.value);
  };

  const handleSubmit = (event: FormEvent<HTMLFormElement>) => {
    event.preventDefault();
    const formData = new FormData(event.currentTarget);
    const theme = formData.get('theme');
    console.log(`Form submitted with theme: ${theme}`);
  };

  return (
    <div>
      <form onSubmit={handleSubmit}>
        <PlusRadioGroup 
          name="theme" 
          label="Select a theme" 
          required 
          onPlusGroupChange={handleChange}
          value={selectedTheme}
        >
          <PlusRadio value="light" text="Light" checked />
          <PlusRadio value="dark" text="Dark" />
          <PlusRadio value="system" text="System" />
        </PlusRadioGroup>
        <div className="result-display">
          Selected theme: <span>{selectedTheme}</span>
        </div>
        <button type="submit">Save Preferences</button>
      </form>
    </div>
  );
};
```

In this example:

1. We use a self-executing anonymous function (`(function(){})()`) to avoid polluting the global scope
2. We select all required DOM elements using proper selectors
3. The button now has a type="button" to prevent form submission and uses an event listener instead of an inline onclick
4. We've added null checks for each DOM element to ensure code robustness

This approach creates a more self-contained example that doesn't rely on global functions, making it more likely to work correctly in documentation environments.

## API

### Plus-Radio

| Property  | Description                                      | Type     | Accepted Values        | Default   |
| --------- | ------------------------------------------------ | -------- | ---------------------- | --------- |
| size      | Sets the size of the radio button                | String   | `sm`, `md`, `lg`       | `md`      |
| text      | Text label displayed next to the radio           | String   | Any valid string       | `null`    |
| checked   | Indicates if the radio button is selected        | Boolean  | `true`, `false`        | `false`   |
| error     | Displays the radio in an error state             | Boolean  | `true`, `false`        | `false`   |
| disabled  | Makes the radio button non-interactive           | Boolean  | `true`, `false`        | `false`   |
| readonly  | Makes the radio read-only but still visible      | Boolean  | `true`, `false`        | `false`   |
| name      | Name attribute used for form submission          | String   | Any valid string       | `''`      |
| required  | Indicates if the radio is required               | Boolean  | `true`, `false`        | `false`   |
| value     | Value submitted with forms when selected         | String   | Any valid string       | `''`      |
